# 项目级工作说明

## 项目目标

本项目是一个基于 Windows 的轻量桌面便签软件。第一版重点是让用户快速记录普通事项和 to do 事项，并按日期归类管理。

## 开发前必读

每次开始开发前，先阅读以下文件：

- `docs/requirements.md`：产品需求、功能范围和不做的内容。
- `docs/development-steps.md`：分阶段开发顺序和每阶段验收目标。
- `docs/data-model.md`：记录、to do、日期归类和逾期滚动的数据结构。
- `docs/design-guidelines.md`：白色、天蓝色、毛玻璃风格的 UI 规范。
- `docs/technical.md`：技术栈、架构和本地存储方案。

## 开发日志要求

- 每天的开发日志放在 `dev-logs/YYYY-MM-DD.md`。
- 每次完成开发后都要更新当天日志。
- 日志必须包含：
  - 已完成
  - 待办事项
  - 问题/风险
  - 下一步

## 执行原则

- 小步推进，每个阶段先完成可验证目标，再进入下一阶段。
- 不跨阶段堆功能。当前阶段之外的想法写入待办，不直接实现。
- 修改用户体验、数据结构、技术方案前，先更新对应文档。
- 保持项目轻量，第一版不做账号、云同步、定时提醒或复杂统计。

## 编码与文档规范

- 文档文件名使用英文，正文可以使用中文。
- 生成或处理中文内容时使用 UTF-8，避免 Windows 管道编码导致中文乱码。
- 面向 Windows 使用场景验证功能，尤其注意中文输入、保存、搜索和重启恢复。
- 如需要下载依赖或外部资源，优先放在 `Z:` 盘新建文件夹中。
